# 사용자 입장에서 보는 챗봇/AI 기능

이 프로젝트에서 사용자가 체감하는 "대화형" 기능은 크게 2가지 챗봇 + 1가지 AI 기능으로 볼 수 있습니다.

## 1) 앱 설명/도움말 챗봇(Docs Bot)
- 역할: "이 앱에서 뭐가 가능해?", "어떤 기능이 있어?", "어떻게 써?" 같은 질문에 답합니다.
- 특징: 이 프로젝트의 지식베이스 문서(kb/*.md)에 있는 내용만 근거로 답하려고 합니다.
- 예시 질문
  - "내 채팅방은 어디서 봐?"
  - "투표는 어떻게 만들어?"
  - "여행 계획 AI는 뭘 참고해서 만들어?"

## 2) 가상 여자친구 챗봇(여름이)
- 역할: 사용자가 메시지를 보내면 '여름이' 페르소나로 대화합니다.
- 사용: `POST /api/girlfriend/chat`
- 예시 질문
  - "여름아 오늘 뭐 먹을까?"
  - "부산 여행 코스 추천해줘"

## 3) AI 여행 계획(키워드/플랜 생성)
- 역할: 채팅방 대화 내용과 키워드, 여행 기간/그룹 취향을 종합해 여행 계획을 생성합니다.
- 흐름(요약)
  - (선택) 채팅 히스토리 → 키워드 추출
  - 키워드 + 기간 + 그룹 프로필 + 관광 API 검색 결과 → 여행 계획 생성
  - 마음에 들면 확정 저장/수정/삭제 가능

## 헷갈릴 때 기준
- "앱 사용법"을 물어보면: Docs Bot
- "그냥 대화/감정형 답변"을 원하면: 여름이
- "구체적 일정표/코스"가 필요하면: AI 여행 계획
